Benchmark coverage matrix

Which memory and agent benchmarks each system has reported scores on, and how those scores compare. Memory papers split into two camps: a memory-specific bench family (LongMemEval, LoCoMo, BABILong, ConvoMem, RULER, MemoryAgentBench, NIAH) that tests recall across sessions and long contexts, and a domain-specific family (GAIA, SWE-bench, WebArena, OSWorld, AIME) that tests downstream task performance where memory is one ingredient among many. Coverage matters because a benchmark with only a handful of reporters can't drive head-to-head comparison — and the memory-specific family is genuinely under-adopted relative to its design intent.

How scores are extracted

  1. Source priority: cells.perf.value first (the curated headline); cells.claims.value only when perf doesn't mention the benchmark. Cells marked as no-data or sentinels like "no public benchmark scores found" are skipped.
  2. Canonicalisation: aliases collapse to one name — e.g. LMES, LME, LongMem-Eval, and LongMemEval-S all become LongMemEval; SciWorld becomes ScienceWorld; BABI-Long becomes BABILong.
  3. Score-shape priority: (1) unit-suffixed in left window (92.2%, +18.7pp, 0.91 F1); (2) unit-suffixed in right window; (3) bare decimal in perf only (0.700 LoCoMo); (4) short bare int adjacent in perf only (30 ConvoMem). Claims mode rejects bare numbers because they're usually pass@k, k-shot, or version-number noise rather than scores.
  4. Parser self-report: 93% of perf mentions yielded a parseable score (69/74); 14% of claims mentions did (7/50). Cells without a score still get a presence ✓.

What to make of this

  • LoCoMo and LongMemEval are the de-facto memory leaderboards. If you want to compare a new memory system head-to-head against shipping work, these are the two benchmarks you'd run first. ByteRover, Mastra, MemPalace and Letta/MemGPT all crowd the top of the LongMemEval table — but absolute scores vary widely (74% → 96%+), so context-window and retrieval-method footnotes matter hugely.
  • ConvoMem is the under-adopted one. If only 1–2 systems report on a benchmark, it can't drive cross-system comparison — it's effectively a single-paper artefact. ConvoMem shows up almost exclusively in Mem0's own reporting, which is why it's flagged "too narrow" above.
  • Memory papers benchmark on their target domain. A paper proposing a memory system for web agents (Mind2Web, WebArena) usually reports on its target domain rather than on a memory-specific axis like LongMemEval. The "both families" bucket is small relative to "domain-only", which is the user's recurring observation about evaluation discipline in the field.
  • Tier 1 systems mostly cluster on LoCoMo + LongMemEval. Filter to T1 only to see this directly — the commercial vendors (Mem0, Zep, Letta, ByteRover, Supermemory, Mastra) all converge on the same two scores. That convergence is what makes those two benchmarks the credible comparison axis.
