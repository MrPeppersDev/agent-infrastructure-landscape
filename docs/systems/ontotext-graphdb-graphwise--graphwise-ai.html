When to use

Optimised for: relationship modeling + reasoning + governance over pure vector

Anti-fit: not for purely-vector or simple-RAG use cases (graph adds setup cost)

Pros & cons

Pros

Best-in-class RDF / SPARQL engine with native reasoning; long-running European / public-sector adoption.

Cons

Property-graph workloads are not a strength; smaller user base than Neo4j.

Claims & capabilities

First triplestore to pass both LDBC SNB Interactive (1.5B edges, SF30) and Semantic Publishing benchmarks; merged with Semantic Web Company to form Graphwise (October 2024) — combined customer count >200 across PoolParty + GraphDB
