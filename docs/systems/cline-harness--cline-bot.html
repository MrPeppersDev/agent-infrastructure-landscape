Cline (harness)

https://cline.bot

OSS VS Code extension that runs as a fully agentic coding assistant — plan/act modes, file ops, shell exec, browser-use, MCP. Originally "Claude Dev" (2024); rebranded Cline 2024-Q4. Multi-model: Anthropic, OpenAI, OpenRouter, Bedrock, Vertex. Memory layer = .clinerules and Memory Bank pattern (distinct rows in catalog).

When to use

Optimised for: Plan/Act loop with explicit human approval gates

Anti-fit: VS Code-only; cost-per-run can spike on long sessions

Pros & cons

Pros

Strongest plan-then-act safety pattern in the OSS coding-agent set; Memory Bank pattern (.cline-memory-bank/) influenced wider "memory in markdown" practice.

Cons

VS Code only (no JetBrains, no terminal); per-token costs surprise users on long autonomous runs.

Claims & capabilities

~45k★ GH; one of the most-installed agentic VS Code extensions; multi-model via standard OpenAI-compatible APIs.
