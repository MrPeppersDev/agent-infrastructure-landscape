MemoryLLM (original)

https://arxiv.org/abs/2402.04624

Augments a transformer with a fixed-size 1B-parameter memory pool distributed across all layers; new knowledge is injected by updating the pool parameters during inference, not via context extension. Distinct from M+ (already in landscape) which extends with scalable long-term storage — this is the original fixed-capacity pool.

Pros & cons

Pros

No external retrieval store needed — memory is entirely parametric and intrinsic to the model.

Cons

Fixed pool capacity creates a hard ceiling on storable knowledge; no dynamic growth path without the M+ extension; pool update is sequential, limiting batched knowledge injection speed.

Claims & capabilities

Effective on model editing benchmarks and long-context tests; stable after ~1M sequential updates.
