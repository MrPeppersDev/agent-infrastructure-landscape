# AI Agent Infrastructure Landscape

Comparative catalog of AI agent infrastructure — memory systems, harnesses,
IDEs, frameworks, training platforms, search substrates, vertical agent
products, voice platforms, robotics stacks, and adjacent infrastructure.
**912 records × 85 columns × 528 typed edges.**

- **Live site:** <https://mrpeppersdev.github.io/agent-infrastructure-landscape/>
- **Top-line findings:** [`docs/FINDINGS.md`](docs/FINDINGS.md) — five
  publishable headlines from the v6 catalog (semantic-cache market gap,
  the 91.3% no-benchmark finding, MCP-spec-as-substrate, the
  under-acknowledged bridge node, single-vendor FM concentration). Same
  five as cards at [/findings](https://mrpeppersdev.github.io/agent-infrastructure-landscape/findings).
- **Schema:** [`docs/SCHEMA.md`](docs/SCHEMA.md)
- **Maintenance contract:** [`MAINTAINER.md`](./MAINTAINER.md)

## Data

`data/landscape.json` is the canonical dataset; `data/landscape.edges.json`
holds the typed relationships between records. Released under CC-BY-4.0 and
semver-tagged via GitHub Releases (`data-v1.0.0` onwards). Decoupled from
the web app per [#35](https://github.com/MrPeppersDev/agent-infrastructure-landscape/issues/35)
so the data outlives any single rendering surface. See
[`data/README.md`](./data/README.md) for the schema overview and intended
use. `landscape.html` is a build artefact regenerated by
`scripts/render.py` from the JSON.

Entries were sourced from curated lists (Agent-Memory-Paper-List,
Awesome-GraphMemory), survey papers, benchmark leaderboards (LongMemEval,
LoCoMo, ConvoMem), vendor sites, academic venue pages, and targeted
research-agent sweeps. Claims are vendor-stated unless otherwise marked.
Honest coverage confidence: ~88–92%; known gaps are tracked in
[`PLAN.md`](./PLAN.md).

## Consumption surfaces

**MCP server** — [`mcp/`](./mcp). Local-stdio server exposing the catalog
to any MCP client (Claude Code, Claude Desktop, etc.) via nine read-only
tools (search, get-by-id, edge traversal, coverage stats, comparison,
recent-changes, eval-orphan detection, substrate blast-radius). See
[`mcp/README.md`](./mcp/README.md).

```sh
cd mcp && npm install && npm run build
claude mcp add landscape -- node $PWD/dist/server.js
```

**CLI** — [`cli/`](./cli). Same nine queries as `landscape <subcommand>`.
Text output by default, `--json` or `--csv` for machine-readable. See
[`cli/README.md`](./cli/README.md).

```sh
cd cli && npm install && npm run build
./dist/landscape.js search "memory" --tier 1 --section "Dedicated memory layers"
```

**Web app** — [`web/`](./web). SvelteKit static export deployed to GitHub
Pages on every push to `main` via `.github/workflows/pages.yml`. Build
output lives under `docs/` for Pages.

```sh
cd web && npm install
npm run dev      # http://localhost:5173/
npm run build    # static export → ../docs/
```

## Editing the catalog

`data/landscape.json` is the source of authority (Path A, since
[#68](https://github.com/MrPeppersDev/agent-infrastructure-landscape/issues/68)).
`landscape.html` is a rendered artefact, never edited by hand.

1. Edit `data/landscape.json` — or let the intake-research / section-audit
   bots open a PR. The PR gets a rendered-cell preview comment from
   `.github/workflows/diff-preview.yml`.
2. `make build` — reconciles JSON, rebuilds edges + citation trajectories,
   re-renders `landscape.html`.
3. `make validate` — see below. Fix any failures.
4. Commit `data/landscape.json`, `data/landscape.edges.json`, and
   `landscape.html` together. CI re-runs validation + the Path A
   byte-identity check on push.

New systems can be proposed via the
[`/submit`](https://mrpeppersdev.github.io/agent-infrastructure-landscape/submit)
form or an `intake`-labelled Issue — both auto-research into a draft PR via
`.github/workflows/intake-research.yml`. See [`docs/INTAKE.md`](docs/INTAKE.md).
Existing rows are periodically re-audited section-by-section by
`.github/workflows/audit-section.yml`. See [`docs/AUDIT.md`](docs/AUDIT.md).

`make refresh-citations` re-pulls Semantic Scholar data (~15 min); only
needed when new research-paper rows are added.

## Validation

`make validate` runs five offline gates in ~25 seconds:

| # | Gate                        | Catches                                                                                          |
|---|-----------------------------|--------------------------------------------------------------------------------------------------|
| 1 | JSON schema                 | Records or edges violating `docs/SCHEMA.md` §7 (bad enums, missing primary, dangling edges).     |
| 2 | Fast-step determinism       | Non-determinism in `extract.py` / `reconcile.py` / `build_edges.py`.                             |
| 3 | Render-cycle stability      | Markup drift in `render.py` ↔ `extract.py` round-trips beyond the 16-line ceiling.               |
| 4 | S2 cache integrity          | Corrupted `extraction/s2-cache/*.json` files.                                                    |
| 5 | Tier-provenance + freshness | Cells whose tier disagrees with their citation, or rows that fail SCHEMA.md §3a/§3b/§7.1.12.     |

`make build-path-a` is a separate gate that asserts `render.py` output is
byte-identical to the committed `landscape.html`. A pre-commit hook is
available via `make install-hooks` (idempotent; short-circuits on
non-pipeline file changes). CI runs both on every PR.

A weekly staleness check (`.github/workflows/staleness.yml`, Mondays
12:00 UTC) flags rows whose upstream repo has gone quiet beyond the
freshness SLA in [`MAINTAINER.md`](./MAINTAINER.md) §2. Run
`make stale-check` for the same scan offline.

## Governance

This catalog has an explicit maintenance contract —
[`MAINTAINER.md`](./MAINTAINER.md). It defines scope, freshness SLAs, the
3-tier claim-validation schema (T1 auto-verifiable → T3 estimate),
succession plan, and contribution channels. Adopted to close the
catalog-deaths-research-identified governance vacuum that killed
DB-Engines, State of JS, and dbdb.io. Three automated workflows enforce
it: intake research, section audit, and weekly staleness scan (see
[`docs/INTAKE.md`](docs/INTAKE.md), [`docs/AUDIT.md`](docs/AUDIT.md),
and the staleness workflow above).
